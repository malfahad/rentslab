# Data models — column reference

Companion to `first draft.txt`. Table names use **snake_case** (PostgreSQL-style). Types are indicative; adjust for MySQL/SQLite (e.g. `JSON` vs `JSONB`, `DATETIME` vs `TIMESTAMPTZ`).

**Conventions**

* Primary keys: `BIGSERIAL` / `BIGINT` auto-increment, or `UUID` if you prefer opaque IDs (document choice once).
* Money: `NUMERIC(14, 2)` (currency minor units; scale up if you need more precision).
* Org scoping: every tenant-facing row either has `org_id` or is reachable only through an FK that carries org (see invariants in `first draft.txt` §3).
* Optional audit fields (recommended on all transactional tables): `created_at`, `updated_at` (`TIMESTAMPTZ NOT NULL` with defaults); optional `created_by_user_id`, `updated_by_user_id` FK → `user`.

**Addresses and geography**

* **`country_code`**: `CHAR(2)` — ISO 3166-1 alpha-2 (e.g. `KE`, `NG`, `GB`).
* **`region`**: state / province / county (label as fits jurisdiction).
* **`postal_code`**: ZIP / postal code (format varies by country).
* Repeatable pattern: `address_line1`, `address_line2` (suite, floor), `city`, `region`, `postal_code`, `country_code`.
* Use `contact_info` JSONB only for overflow or legacy imports when you already have structured columns.

**Navigation map (staff UI)**

| Section | Screens | Primary tables |
|---------|---------|----------------|
| Portfolio | Landlords, Buildings, Units | `landlord`, `building`, `unit` |
| Operations | Tenants, Lease Arrangements, Services | `tenant`, `lease`, `service`, `service_subscription` |
| Finance | Invoices, Credit Notes, Payments, Expenses | `invoice`, `invoice_line_item`, `credit_note`, `payment`, `payment_allocation`, `expense_category`, `expense` |
| Maintenance | Job Orders, Vendors | `job_order`, `vendor` |

Shared cross-cutting: `org`, `user`, `user_role`.

## How address fields work together

* **Building** holds the canonical physical site address; **unit** adds `floor`, `entrance`, and optional `address_override_*` when the postal address differs.
* **Tenant** holds default correspondence and company identifiers.
* **Lease** can override billing address via `billing_same_as_tenant_address` + `billing_*` when notices and invoices must go elsewhere.
* **Invoice** `bill_to_*` columns are **snapshots at issue time** (copy from tenant/lease/billing rules when the invoice is generated) so PDFs and legal copies stay stable if the tenant moves later.
* **Vendor** mirrors **landlord**-style address/contact columns for payables and job correspondence.

## Invariant checklist (implementation)

| Rule | Check |
|------|--------|
| Landlord / building | `building.org_id = landlord.org_id` for `building.landlord_id` |
| Lease | `tenant.org_id =` org of `unit.building` |
| managed_by | If `lease.managed_by_user_id` set, exists `user_role` with same `org_id` as building |
| Payment org | `payment.org_id = tenant.org_id` |
| Payment lease_id | If set, all allocations’ invoices have `invoice.lease_id = payment.lease_id` |
| Service subscription | `service.org_id` = org of lease (via unit → building) |
| Invoice org (if used) | `invoice.org_id` = same org as lease’s building |
| Vendor | `vendor.org_id` consistent with any linked `job_order` / `expense` |
| Job order | `job_order.org_id` = `building.org_id`; `unit_id` ∈ that building |
| Expense | `expense.org_id` matches `building`, `lease`, `vendor`, `job_order` when those FKs are set |
| Expense category | `expense_category.org_id = expense.org_id` for `expense.expense_category_id` |
